# Case Study — End-to-End RAG 검색·추천 시스템: 의도 분류부터 랭킹까지

> 국내 생활가전 제조사 | ML Engineer (검색·랭킹 노드 전담 개발) | 2025.03 – 2025.09
> ※ 고객사 보안을 위해 구체 정보는 일반화했으며, 모든 다이어그램·수치는 재구성한 것입니다.

## 1. 배경

제조 현장에서 품질 이슈가 발생하면 담당자가 과거 유사 사례와 해결책을 수작업으로 찾고 있었다. 목표는 사내 메신저(Teams) 챗봇으로 "과거 유사 품질문제와 해결책을 검색·추천"하는 시스템을 만드는 것.

## 2. 검색 퍼널 설계

단일 벡터 검색으로는 부족했다. 사용자 질의가 "특정 부품 키워드 검색"부터 "증상 서술형 질문"까지 다양했기 때문이다. LangGraph로 검색 퍼널을 단계별 노드로 분리했다.

```
사용자 질의
   → [의도 분류]  질의 유형 판별 (Few-shot + CoT)
   → [검색]       유형별 인덱스 라우팅, 하이브리드 검색
   → [랭킹]       비즈니스 우선순위 정렬
   → 응답 생성
```

**의도 분류 (query understanding)** — system prompt에 의도-질의 Few-shot 예시 쌍을 넣고, 분류 근거를 함께 출력하게 하는 CoT를 적용해 분류 품질을 높였다. LangGraph 상태 관리로 분류 결과에 따라 이후 노드의 컨텍스트를 동적으로 주입했다.

**검색 (retrieval)** — 용도별 인덱스 3종(키워드·클래스·전문 검색)을 구축했다. 임베딩은 text-embedding-3-large, ANN은 HNSW 파라미터를 튜닝했다.

**랭킹 (ranking)** — 검색 유사도만으로는 현업의 "중요한 사례 먼저"라는 요구를 충족할 수 없었다. 중요도·최신성(일자)·수용률 시그널을 결합한 비즈니스 우선순위 정렬 로직을 설계·구현했다. (전담 영역)

## 3. 검색 품질 실험

"어떤 검색 방식이 실제로 정확한가"를 감이 아니라 실험으로 결정했다.

| 비교 대상 | 방식 |
|---|---|
| Simple | 키워드 매칭 |
| Semantic | 시맨틱 재순위화 |
| Full | 전문 검색 |
| Hybrid | 벡터 + 키워드 결합 |

동일 질의 셋으로 모드별 검색 결과를 비교 평가한 결과, 하이브리드 구성이 최적이었다. 코사인 유사도 기반 시맨틱 검색은 기존 키워드 시스템이 놓치던 "표현은 다르지만 같은 증상" 사례를 잡아냈다.

## 4. 위기 대응: 무중단 인덱스 재설계

프로젝트 중반, 고객사의 원천 데이터 체계가 변경됐다. 약 1개월간 인덱스 재설계와 데이터 재적재를 주도해 서비스 중단 없이 마이그레이션을 완수했다.

## 5. 운영

- 프롬프트를 소스 코드와 동일하게 Azure DevOps로 버전 관리 — 실험 재현성 확보
- AKS 기반 CI/CD, kubectl로 데이터 적재 모니터링·로그 분석
- FastAPI + Streamlit 서빙

## 6. 결과

- 담당 모듈 UAT 100% 통과 (전체 96%+)
- 고객 정의 KPI(업무 절감시간) 달성 — 수동 검색 프로세스가 챗봇 자동화로 대체
- 시맨틱 검색으로 기존 키워드 검색 대비 정확도 대폭 개선

## 7. 배운 점

- 검색 품질은 단일 모델이 아니라 퍼널 전체(질의 이해 → 검색 → 랭킹)의 곱이다. 가장 큰 개선은 랭킹 레이어에서 나왔다 — 사용자가 원하는 건 "유사한 문서"가 아니라 "지금 봐야 할 문서"였다.
- 프롬프트도 코드다. 지속적인 전 관리 없이는 "지난주엔 됐는데 왜 안 되는 거지?를 디버깅할 수 없다.
